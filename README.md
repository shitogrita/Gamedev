# Gamedev
# BrickGame Portfolio

## О проекте

В данном репозитории представлены два проекта, реализующих классические игры BrickGame с использованием разных подходов к разработке:

- **BrickGame Tetris** — реализация на языке **C (C11)** с применением структурного программирования;
- **BrickGame Snake** — реализация на языке **C++ (C++20)** с использованием объектно-ориентированного подхода.

Оба проекта основаны на формализации игровой логики через **конечный автомат (FSM)** и разделении логики и интерфейса.

---

# Проект 1. BrickGame Tetris

## Общее описание

Реализована игра **Тетрис** с разделением на:

- библиотеку, реализующую игровую логику;
- терминальный интерфейс на базе `ncurses`.

Игровая логика построена на основе конечного автомата, который описывает состояния игры и переходы между ними.

---

## Реализованный функционал

### Основная игровая механика
- генерация всех типов фигур (тетрамино);
- вращение фигур;
- перемещение по горизонтали;
- ускоренное падение;
- мгновенное падение (hard drop);
- обработка столкновений;
- фиксация фигуры при достижении границы или других фигур;
- определение заполненных линий;
- удаление заполненных линий;
- смещение верхних блоков вниз;
- завершение игры при достижении верхней границы;
- отображение следующей фигуры.

---

### Управление и интерфейс
- терминальный интерфейс с использованием `ncurses`;
- поддержка управления:
  - старт игры;
  - пауза;
  - завершение;
  - движение влево/вправо;
  - ускорение падения;
  - вращение фигуры.

---

### Дополнительные механики
- система подсчёта очков;
- сохранение рекорда между запусками;
- система уровней;
- увеличение скорости с ростом уровня.

---

## Техническая реализация

### Используемые технологии
- C11
- gcc
- ncurses
- Check (unit-тестирование)
- Makefile
- gcov (анализ покрытия)

---

### Архитектурные особенности
- использование конечного автомата (FSM);
- разделение логики и интерфейса;
- реализация в виде библиотеки с чётким API;
- покрытие тестами не менее 80%;
- соответствие Google Style.

---

## FSM в проекте

Игровой процесс представлен в виде состояний:

- `Start`
- `Spawn`
- `Moving`
- `Move`
- `Attaching`
- `GameOver`

Переходы между состояниями происходят либо по таймеру, либо при действиях пользователя.

---

## Результат

Реализована библиотека игровой логики и полноценный терминальный интерфейс, поддерживающий весь игровой цикл, включая дополнительные механики.

# Проект 2. BrickGame Snake

## Общее описание

Реализована игра **Snake** на языке C++20 с использованием:

- объектно-ориентированного подхода;
- архитектуры **MVC**;
- конечного автомата для управления состояниями игры.

Проект состоит из двух частей:

- библиотека с игровой логикой;
- десктопный GUI (Qt/GTK+).

Также реализована совместимость:
- с консольным интерфейсом BrickGame v1.0;
- с десктопным интерфейсом для Tetris.

---

## Реализованный функционал

### Основная игровая механика
- автоматическое движение змейки по таймеру;
- управление направлением (влево/вправо относительно текущего движения);
- генерация «яблок» на игровом поле;
- увеличение длины змейки при поедании яблока;
- завершение игры при:
  - столкновении со стеной;
  - столкновении с самой собой;
- победа при достижении длины 200;
- ускорение движения по действию пользователя;
- стартовая длина змейки — 4 клетки;
- поле размером 10x20.

---

### Дополнительные механики
- подсчёт очков (1 яблоко = 1 очко);
- сохранение рекорда между запусками;
- обновление рекорда в реальном времени;
- система уровней:
  - каждые 5 очков → новый уровень;
  - увеличение скорости;
  - максимум 10 уровней.

---

## Архитектура

### MVC
- **Model** — библиотека с логикой игры (FSM, состояние, правила);
- **Controller** — обработка пользовательского ввода;
- **View** — графический интерфейс (без бизнес-логики).

---

### Особенности реализации
- использование пространства имён `s21`;
- применение RAII;
- разделение на библиотеку и GUI;
- отсутствие бизнес-логики в интерфейсе;
- тонкий контроллер;
- покрытие тестами не менее 80% (GTest);
- соблюдение Google Style;
- сборка через Makefile.

---

## FSM в проекте

Игровая логика представлена как конечный автомат со следующими состояниями:

- ожидание старта;
- движение;
- обновление состояния по таймеру;
- обработка столкновений;
- завершение игры;
- победа.

FSM используется для формального описания поведения игры и контроля переходов между состояниями.

---

## Результат

Реализована модульная библиотека игровой логики Snake, пригодная для повторного использования, а также графический интерфейс, полностью поддерживающий игровой процесс и дополнительные механики.
